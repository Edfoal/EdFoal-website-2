1. Problem Statement & Current State

The website's primary navigation items were defined independently across three separate files. Despite the existence of a dedicated data file (src/data/navigation.ts) intended to act as the single source of truth, it was never imported anywhere in the codebase. Instead, both the header navigation (Navbar.tsx) and footer navigation (LargeNameFooter.tsx) relied on ad-hoc, hardcoded inline arrays.

This isolation led to label drift and UX inconsistency across different sections of the website.

2. Root Cause Analysis

⚠ Architectural Drift & Dead Code
When the layout components (Navbar and LargeNameFooter) were developed or iterated upon, navigation links were hardcoded directly inside the component scopes for speed. The original data dictionary (src/data/navigation.ts) was abandoned and became dead code, creating a false impression of centralized configuration.
  1. Lack of Centralized Enforcement: No linting rule or architectural constraint prevented developers from defining hardcoded navigation arrays inside UI components.
  2. Iterative Copy Alterations: Over time, individual developers tweaked copy in the footer (adding "Our" to Services) or header (changing "Case Studies" to "Usecases") without realizing a central schema existed.

3. Business & Technical Impact

  • Inconsistent User Experience: Users see "Services" at the top of the page but "Our Services" at the bottom. The route /usecases is labeled "Case Studies" in the data dictionary but "Usecases" in the UI, causing potential confusion regarding site hierarchy.
  • Maintenance Overhead: Adding a new page, changing a URL slug, or updating a navigation label requires hunting down and editing multiple files, increasing the risk of broken links or missed updates.
  • Dead Code Bloat: src/data/navigation.ts added unnecessary cognitive load for developers who assumed editing it would update the website navigation.

4. Remediation Implemented

We established src/data/navigation.ts as the authoritative single source of truth, standardizing the labels across the entire application and eliminating inline arrays.
