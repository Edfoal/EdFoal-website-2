1. Problem Statement & Current State

During a frontend performance and Core Web Vitals audit, it was identified that raw HTML <img> tags are used extensively across 38+ locations in the codebase, while the optimized Next.js <Image /> component is used in only one file (container-scroll-animation.tsx).

Furthermore, the application's Next.js configuration (next.config.ts) contains an empty configuration object with no image domains or remote patterns defined: 

// Current state of next.config.ts — No remote patterns!
const nextConfig: NextConfig = {
  /* config options here */
};

In Next.js 15 App Router, using raw <img> tags bypasses the built-in Next.js Image Optimization API. As a result, images are served without automatic WebP/AVIF format conversion, responsive resizing (srcset), or native lazy loading. This triggers over 20+ @next/next/no-img-element ESLint build warnings/errors and severely degrades Largest Contentful Paint (LCP) scores on mobile devices.

2. Root Cause Analysis

🚨 Missing CDN Remote Pattern Whitelisting

When developers attempted to use external assets hosted on ImageKit (https://ik.imagekit.io/...) and Unsplash (https://images.unsplash.com/...), Next.js threw runtime security exceptions because those hostnames were not whitelisted in next.config.ts. Instead of configuring images.remotePatterns, developers reverted to raw HTML <img> tags across the entire frontend to bypass the security check during rapid prototyping.

  • Unconfigured Security Whitelisting: Next.js requires explicit hostname authorization via remotePatterns in next.config.ts before its image server will fetch and optimize external CDN assets. Because this configuration was missing, <Image /> failed at runtime for external URLs.
  • Rapid UI Scaffolding & Component Paste: When integrating third-party UI sections and marketing cards, raw HTML templates containing standard <img> tags were pasted directly into React components without converting them to Next.js <Image /> primitives.
  • Ignored Linter Warnings: During local development, @next/next/no-img-element ESLint rules were treated as non-blocking warnings rather than hard architecture constraints, allowing unoptimized image usage to proliferate across 38+ locations.

3. Business & Technical Impact

  • Severe LCP Degradation (SEO Penalty): Unoptimized hero banners, logos, and case study graphics download as multi-megabyte PNGs/JPEGs. This delays Largest Contentful Paint (LCP) times beyond the Google Search 2.5-second threshold, directly harming Google PageSpeed SEO rankings and user conversion rates.
  • Wasted Mobile Bandwidth: Without responsive srcset generation or sizes attributes, a mobile phone with a 375px screen is forced to download 3000px wide desktop banners, wasting cellular data and causing noticeable visual stuttering during page rendering.
  • Thread Blocking & Missing Lazy Loading: Raw <img> tags default to eager loading, causing dozens of off-screen partner logos and footer graphics to download simultaneously on page load, choking browser network queues and delaying Time to Interactive (TTI).
  • Codebase Technical Debt: 20+ ESLint linting violations clutter build reports and obscure legitimate compilation warnings.

4. Remediation Plan

We propose a systematic two-step remediation: "Configure remotePatterns in next.config.ts and replace external <img> tags with <Image fill sizes="..." /> or explicit dimensions."

Step 1: Configure remotePatterns in next.config.ts

Update next.config.ts to securely authorize our external CDN providers (ImageKit, Unsplash, Aceternity) for automatic image optimization:

 import type { NextConfig } from "next";

 const nextConfig: NextConfig = {
-  /* config options here */
+  images: {
+    remotePatterns: [
+      { protocol: "https", hostname: "ik.imagekit.io", pathname: "/edfoalImage/**" },
+      { protocol: "https", hostname: "images.unsplash.com" },
+      { protocol: "https", hostname: "assets.aceternity.com" },
+    ],
+  },
 };

 export default nextConfig;

Step 2: Systematically Refactor Raw <img> to <Image />

Replace raw <img> tags across UI components with Next.js <Image /> components, applying proper sizes, fill, or explicit width/height attributes:

+import Image from "next/image";

// Before (Raw HTML img):
-<img key="bhavishya" src="https://ik.imagekit.io/edfoalImage/assets/image/bhavishyaNGO.png" alt="Bhavishya NGO" className="partner-logo" />

// After (Optimized Next.js Image):
+<Image
+  key="bhavishya"
+  src="https://ik.imagekit.io/edfoalImage/assets/image/bhavishyaNGO.png"
+  alt="Bhavishya NGO"
+  width={140}
+  height={50}
+  className="partner-logo object-contain"
+/>
